# Contributing to Sadgi Protocol

We welcome contributions from the community! Sadgi is designed to be the foundational Trust Layer for the Stellar ecosystem, and it requires a diverse set of contributors to succeed.

## How to Contribute
1. **Fork the Repository**: Clone it locally.
2. **Branch Naming**: Use `feature/issue-number-description` or `fix/issue-number-description`.
3. **Local Setup**: See [README-dev.md](./README-dev.md) for how to run the environment locally.
4. **Testing**: Run `make test` and `make format` before committing.

## Sadgi Improvement Proposals (SIPs)
Major architectural changes or modifications to Protocol Standards (like the `SadgiReceipt`) require a formal RFC. 
Please open an Issue using the `[SIP]` prefix to initiate a discussion before writing code. 

Accepted SIPs are documented in `docs/src/rfcs/`.

## Code of Conduct
Please review our [Code of Conduct](./.github/CODE_OF_CONDUCT.md) before participating in the community.
